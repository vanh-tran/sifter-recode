Gap Analysis

Previous architecture vs. what we agreed to build

Pipeline — Well Covered ✓

PresentAsync pipeline with job queue

Two-phase worker model (OCR+classify → process) is exactly Approach B. Solid.

PresentInvoice detection + structured parsing

Gate on bad parse is a great addition we hadn't discussed — keep it.

PresentFindings taxonomy

Fast checks + Audit Agent split is excellent. The findings table is more specific than needed — that's a good thing.

Gaps — Critical

MissingDispute workflow

Entirely absent. Flag → AI-draft letter → send from app → track status is the core value-add for the AP team. The pipeline produces findings, but there's no workflow for acting on them.

MissingContracted rate sheet audit check

The findings table covers fuel, detention, lumper, accessorials — but not base line-haul rate vs. contracted rate. For FTL, the biggest overcharge is often a wrong per-mile or flat rate. Rate sheets need to be uploaded and compared.

MissingOutlook OAuth intake

Current flow is Gmail-only (via n8n). We agreed on Gmail + Outlook + manual upload.

MissingManual upload intake

No PDF drag-and-drop path. Some AP teams receive invoices by mail or outside email — they need to upload directly.

Gaps — Supporting Data

PartialRate sheet management

The audit agent needs contracted rate sheets to exist in the system, but there's no ingestion or management flow for them. Where do they live? How does the AP team upload and update them per carrier?

PartialCarrier contact management

Needed for the dispute workflow — to send a dispute email, the app needs to know the carrier's billing contact. Not present.

PartialBOL mismatch as explicit audit check

Document linking exists, but "BOL says X, invoice says Y" isn't in the findings table. This should be a named finding type (weight, origin/destination, service type mismatch).

What the Previous Arch Has That's Worth Keeping

KeepGate/abort on bad parse · Dedup findings · near-duplicate invoice detection · n8n orchestration · GCS for files · Supabase for structured data · action_needed/reviewing/cleared status
